Part 1: R & RStudio Basics

What is R? What is RStudio?

  • R: the programming language/engine that does the calculations
  • RStudio: the friendly “cockpit” (IDE) we use to write and run R code
  • Packages: add-on toolkits (e.g. tidyverse for data wrangling, ggplot2 for graphs) — install once, load every session

The RStudio layout

  • Source (top-left): where you write scripts (.R files)
  • Console (bottom-left): where code actually runs
  • Environment (top-right): objects you’ve created (data, values)
  • Files/Plots/Packages/Help (bottom-right)

Live demo: tour of RStudio

R Projects — start here every time

# File > New Project > New Directory > New Project
# Keep your script, data/, and outputs together in ONE folder

Core building blocks

The pipe %>% (or |>)

Reads as “then” — chains steps together instead of nesting functions.

# Without pipe (hard to read)
round(mean(ages), 1)

# With pipe (reads top-to-bottom)
ages %>%
  mean() %>%
  round(1)

We will use the pipe constantly today.

Part 3: Cleaning the Line List

Why real surveillance data is messy

  • Multiple data collectors → inconsistent spelling/casing
  • Manual date entry → mixed formats
  • Weak network / re-submission → duplicate records
  • Skipped fields → missing values
  • Fat-finger errors → impossible values (age = 150)

Cleaning is not optional — it is the analysis.

Step 1: Clean column names

library(janitor)

linelist <- linelist_raw %>%
  clean_names()

names(linelist)

clean_names() turns "Sex ", " Ward", "date of report" into tidy snake_case names: sex, ward, date_of_report.

Step 2: Find & remove duplicates

get_dupes(linelist)              # inspect exact duplicates
linelist <- linelist %>% distinct()

Near-duplicates (e.g. "AWD-0043" vs "AWD-0043b") need a rule:

linelist <- linelist %>%
  mutate(case_id = str_remove(case_id, "[a-z]$")) %>%
  distinct(case_id, .keep_all = TRUE)

Step 3: Clean categorical variables

tabyl(linelist, sex)   # ALWAYS inspect before cleaning
linelist <- linelist %>%
  mutate(sex = case_when(
    str_to_upper(str_trim(sex)) %in% c("M", "MALE")   ~ "Male",
    str_to_upper(str_trim(sex)) %in% c("F", "FEMALE") ~ "Female",
    TRUE ~ NA_character_
  ))

case_when() is your best friend for recoding messy categories.

Step 3 (continued): ward names

tabyl(linelist, ward)
linelist <- linelist %>%
  mutate(
    ward = str_trim(ward),
    ward = str_to_title(ward),
    ward = recode(ward,
                  "Iwo Rd"  = "Iwo Road",
                  "Ring Rd" = "Ring Road")
  )

Pattern: trim whitespace → standardize case → fix known typos.

Step 4: Clean numeric variables (age)

summary(linelist$age)     # spot the impossible max!
linelist <- linelist %>%
  mutate(age = if_else(age < 0 | age > 100, NA_real_, age))

Then create age groups for later analysis:

linelist <- linelist %>%
  mutate(age_group = cut(age,
    breaks = c(0, 5, 15, 30, 45, 60, Inf),
    labels = c("0-4","5-14","15-29","30-44","45-59","60+"),
    right = FALSE))

Step 5: Clean dates — the tricky part

Our data has THREE date formats mixed together:

  • 2026-05-03 (ISO: year-month-day)
  • 03/05/2026 (day/month/year)
  • 03-May-2026 (day-month abbreviation-year)
library(lubridate)

linelist <- linelist %>%
  mutate(date_onset = parse_date_time(
    date_onset, orders = c("ymd", "dmy", "mdy"), quiet = TRUE
  ) %>% as_date())

⚠️ A word of caution on dates

11/05/2026 is genuinely ambiguous: is that 11 May or 5 November?

  • Always ask your data source which format was used
  • Where possible, request ISO format (YYYY-MM-DD) at collection stage
  • After parsing, always sanity-check the range of resulting dates
range(linelist$date_onset, na.rm = TRUE)

Step 5 (continued): validate & fix dates

linelist <- linelist %>%
  filter(!is.na(date_onset),
         date_onset >= as_date("2026-01-01"),
         date_onset <= as_date("2026-07-04")) %>%
  mutate(
    date_of_report = if_else(
      date_of_report < date_onset | is.na(date_of_report),
      date_onset, date_of_report),
    reporting_delay = as.numeric(date_of_report - date_onset)
  )

Save your cleaned dataset

write_csv(linelist, "data/awd_linelist_clean_inclass.csv")
